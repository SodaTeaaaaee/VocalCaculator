# Vocal Calculator Subagent Protocol

本文档规定执行模型在本项目中如何强制使用子 agent。

目标：

1. 提高并发效率
2. 降低主上下文污染
3. 让较弱模型减少一次性吞下过多问题

## 1. 总原则

主 agent 不得单线程硬做整个项目。

每到一个非平凡阶段，都必须先判断哪些任务可以并发委派。

子 agent 必须只做：

1. 一个边界清晰的问题
2. 一个有限写入范围的实现任务
3. 一个独立验证任务

禁止：

1. 让多个子 agent 修改同一组文件
2. 让子 agent 同时探索和重写整个架构
3. 把最关键且立即阻塞的主路径完整丢给子 agent 后自己空等

## 2. 强制并发阶段

### 阶段 A：开工前扫描

至少并发派出 3 个子 agent：

1. 代码与目录现状扫描
2. 资源与音频资产扫描
3. 构建环境与平台依赖扫描

### 阶段 B：研究与核验

至少并发派出 4 个子 agent：

1. GUI / 平台资料核验
2. 音频栈核验
3. Android 构建与签名核验
4. 第三方资产与许可核验

### 阶段 C：实现期

根据写边界拆分为互不重叠任务，例如：

1. 纯计算核心与测试
2. Slint UI 壳层与布局
3. 音频注册表与播放控制
4. 资源映射实现（优先硬编码表）
5. 构建脚本与 Android 配置

### 阶段 D：验收期

至少并发派出 3 个子 agent：

1. 逻辑测试与边界检查
2. UI/交互回归检查
3. 文档、归因、第三方说明检查

## 3. 每个子 agent 的硬性输出

每个子 agent 完成时必须产出两件事：

1. 对主 agent 的摘要回复
2. 一个同步文件到 `.agents/sync/`

同步文件命名规范：

`YYYY-MM-DD-<phase>-<topic>.md`

示例：

1. `2026-06-08-research-audio-kira.md`
2. `2026-06-08-impl-core-percent-mu.md`
3. `2026-06-08-verify-android-build.md`

## 4. 子 agent 同步文件模板

每个同步文件必须包含：

1. 任务范围
2. 输入前提
3. 关键发现
4. 改动文件列表
5. 未解决问题
6. 建议主 agent 的下一步

## 5. 推荐委派模板

### 5.1 研究类模板

要求子 agent：

1. 只查一手来源
2. 给出明确结论，不要模糊
3. 标出来源 URL
4. 说明哪些是事实，哪些是推断

### 5.2 实现类模板

要求子 agent：

1. 只改指定文件范围
2. 完成后列出改动文件
3. 说明运行了哪些验证命令
4. 若未验证，要明确说未验证

### 5.3 验证类模板

要求子 agent：

1. 不重写实现
2. 只做复核、测试与风险检查
3. 发现问题时给最短可执行修复建议

## 6. 主 agent 的整合责任

主 agent 不能只转发子 agent 结论。

主 agent 必须：

1. 合并冲突结论
2. 拒绝未经核实的猜测
3. 把最终决策写回主文档或同步文件

## 7. 子 agent 何时必须联网

下列情形必须联网核实：

1. 库版本
2. 官方平台支持
3. 许可条款
4. Android 构建链
5. 打包与发布规则
6. 任何“最新”或“当前”信息

## 8. 停止条件

如果主 agent 发现：

1. 子 agent 给出互相冲突的结论
2. 子 agent 修改边界重叠
3. 子 agent 在无授权前提下默认分发第三方资产

则必须中止继续扩散实现，先收敛分歧。
